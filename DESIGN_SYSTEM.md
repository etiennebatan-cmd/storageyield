# StorageYield Design System

## Brand
- Name: StorageYield
- Tone: serious, operator-first, Benelux-native
- Target: self-storage, garagebox, containeropslag operators in NL/BE

## Typography
- Headings: `font-semibold`, `tracking-tight`
- Body: `text-base` / `text-lg`, `leading-7`
- UI labels: `text-sm`, `font-semibold`

## Layout
- Page width: `max-w-[1240px]`
- Section padding: `px-5 py-14`
- Cards: `rounded-3xl border border-slate-200 bg-white shadow-sm`
- Buttons:
  - Primary: `rounded-full bg-slate-950 text-white`
  - Secondary: `rounded-full border border-slate-200 bg-white text-slate-950`

## Components
- `MarketingNav`: sticky header with product, oplossingen, Benelux, vergelijk, contact groups
- `MarketingFooter`: site links and brand summary
- `ProductMockup`: simple status panel previews for key modules
- `ProcessFlow`: operator workflow steps with status badges
- `FeatureStatusSwitcher`: live/pilot/roadmap switcher for product transparency

## Content guidance
- Avoid vague platform names such as "architecture" or "lifecycle" as core headings.
- Keep page copy concrete: booking flows, access workflows, pricing decisions, Benelux payment readiness.
- Use clear status language: `Beschikbaar`, `Pilot`, `Roadmap`.
- Do not overclaim integrations; describe current capability and planned roadmap.
